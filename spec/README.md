# Spec folder

Living specifications for this app. Start with [`prompts/03-spec-driven-development.md`](../prompts/03-spec-driven-development.md) to generate these docs for your product.

| Document | Purpose |
| -------- | ------- |
| `STACK.md` | Technology stack and included flows (already populated for the template) |
| `PRODUCT_BRIEF.md` | North star — what, why, who, v1 scope |
| `FEATURE_SPEC.md` | User stories, build order, screens, acceptance criteria |
| `API_CONTRACT.md` | Request/response shapes clients rely on |
| `BUSINESS_RULES.md` | Product logic that spans multiple endpoints |
| `PROJECT_OVERVIEW.md` | What this repo actually does as it evolves |
| `tasks/` | Numbered, agent-executable implementation slices |
